## Distributed UAT Broker Topology

The distributed UAT broker topology requires a Nova-hosted dedicated Nexus NATS endpoint reachable from Jarvis, for example `nats://192.168.31.124:7422`.

Use this topology only after a later runtime/UAT gate:

- Set `NATS_URL=nats://192.168.31.124:7422`.
- Set `NEXUS_RESIDENT_CONTROLLER_NATS_URL=nats://192.168.31.124:7422`.
- Set auth through an approved redacted auth reference such as `NEXUS_RESIDENT_CONTROLLER_NATS_AUTH_REF`, never inline credentials.
- Run firewall preflight from Jarvis to Nova host and confirm TCP reachability to port `7422`.
- Run auth preflight with the approved token/ref and record non-secret evidence.
- Run subject allowlist preflight before any runtime start.

Broker no-go rules:

- never OpenClaw 4222.
- never Jarvis-side 127.0.0.1.
- never `nats://127.0.0.1:4222`.
- never production/default broker port `4222` for distributed UAT.
- never mutate broker config, credentials, or firewall rules without explicit Alex/Nova approval.
