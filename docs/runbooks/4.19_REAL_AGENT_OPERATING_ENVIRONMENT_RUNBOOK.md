## Distributed UAT Broker Topology

The distributed UAT broker topology requires a Nova-hosted dedicated Nexus NATS endpoint reachable from Jarvis, for example `nats://192.168.31.124:7422`.

Use this topology only after a later runtime/UAT gate:

- Set `NATS_URL=nats://192.168.31.124:7422`.
- Set `NEXUS_RESIDENT_CONTROLLER_NATS_URL=nats://192.168.31.124:7422`.
- Set auth through an approved redacted auth reference such as `NEXUS_RESIDENT_CONTROLLER_NATS_AUTH_REF`, never inline credentials.
- Run firewall preflight from Jarvis to Nova host and confirm TCP reachability to port `7422`.
- Run auth preflight with the approved token/ref and record non-secret evidence.
- Run subject allowlist preflight before any runtime start.

Broker no-go rules:

- never OpenClaw 4222.
- never Jarvis-side 127.0.0.1.
- never `nats://127.0.0.1:4222`.
- never production/default broker port `4222` for distributed UAT.
- never mutate broker config, credentials, or firewall rules without explicit Alex/Nova approval.

## Source-Only Implementation Flow

1. Pin branch, head, base, and merge-base.
2. Confirm Candidate Adapter dependency head is present.
3. Run focused tests for Candidate Adapter policy, lifecycle, heartbeat, eligibility/reservation, dispatch guard, evidence package, Runbook, A2A placeholder, and readiness taxonomy.
4. Run regression slices for resident-controller, dispatch eligibility, heartbeat, registry, candidate runtime, message contracts, private-agent boundaries, and Agent Access projection.
5. Run `python -m compileall -q nexus/mq`.
6. Run `git diff --check`.
7. Run full `python -m pytest nexus/mq/tests -q`.
8. Generate evidence package and SHA256 manifest.
9. Run high-confidence secret scan.
10. Return `READY_FOR_NOVA_REVIEW` or `BLOCKED`.

## Assignment Preconditions

Assignment publish requires:

- source-bound dispatch request
- allowed lifecycle decision id
- active reservation lease id
- matching assignment id
- matching runtime instance id
- matching dispatch run id
- idempotency key
- no-go scope ref
- non-business assignment kind

Heartbeat freshness, route readiness, controller init, MQ ACK, or MiniTest diagnostic evidence cannot substitute for lifecycle decision plus active lease.
